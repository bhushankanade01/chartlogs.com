## ⚙️ BACKEND & DATABASE

### Database Schema (key tables):
- users (id, email, name, password_hash, plan, timezone, currency, created_at)
- trades (id, user_id, symbol, type, entry, exit, size, sl, tp, open_time, close_time, pnl, fees, source, tags, emotion, notes, screenshots, created_at)
- journal_entries (id, trade_id, user_id, notes, screenshots, mood, created_at)
- broker_connections (id, user_id, broker_type, credentials_encrypted, status)
- subscriptions (id, user_id, plan, stripe_customer_id, stripe_subscription_id, status, ends_at)

### API Endpoints:
- POST /api/auth/register, /login, /logout, /refresh
- GET/POST/PUT/DELETE /api/trades
- GET/POST/PUT /api/journal/:tradeId
- GET /api/dashboard/stats
- GET /api/analytics/performance?period=30d
- GET /api/market/calendar
- POST /api/ai/report (Pro only)
- POST /api/tools/position-size
- GET /api/subscription/plans
- POST /api/subscription/checkout (Stripe)

### Auto-calculations on trade save:
- P&L = (Exit - Entry) × Size × ContractSize - Fees
- R:R = (TP - Entry) / (Entry - SL)
- Pips = (Exit - Entry) / pip size for that symbol
- Win = P&L > 0
