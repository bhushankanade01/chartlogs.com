CHARTLOGS INTERNAL APP REDESIGN
================================

The landing page looks great now, but the internal dashboard/app needs 
a complete visual overhaul. We're targeting a modern, polished, Bloomberg-like 
aesthetic with smooth animations and better visual hierarchy.

=== COLOR PALETTE (Dark + Light Modes) ===

DARK MODE (Primary):
  Background: #0F172A (deep navy)
  Surface: #1E293B (slightly lighter)
  Card bg: #1E293B with rgba border
  Text primary: #F8FAFC (off-white)
  Text secondary: #94A3B8 (muted gray)
  Accent: #3B82F6 (blue for CTAs)
  Success: #10B981 (green for profits/gains)
  Danger: #EF4444 (red for losses)
  Warning: #F59E0B (orange for alerts)
  Border: #334155 (subtle divider)

LIGHT MODE:
  Background: #F8FAFC
  Surface: #FFFFFF
  Card bg: #FFFFFF with subtle shadow
  Text primary: #0F172A
  Text secondary: #64748B
  Accent: #3B82F6
  (Keep trading colors same)

=== DESIGN SYSTEM ===

SPACING:
  xs: 4px
  sm: 8px
  md: 16px
  lg: 24px
  xl: 32px
  Use consistently across all components

TYPOGRAPHY:
  Headlines (H1): 32px, bold, letter-spacing -0.5px
  Subheadings (H2): 24px, semibold
  Body (p): 14px or 16px, regular
  Labels: 12px, uppercase, letter-spacing 0.5px
  Mono (numbers): 16px, font-family: 'Monaco' or 'Courier New'

BORDER RADIUS:
  Buttons: 8px
  Cards: 12px
  Inputs: 8px
  Large elements: 16px

SHADOWS (Dark Mode):
  Subtle: 0 1px 3px rgba(0,0,0,0.3)
  Medium: 0 4px 12px rgba(0,0,0,0.4)
  Large: 0 12px 32px rgba(0,0,0,0.5)

ANIMATIONS:
  Hover transitions: 200ms ease-in-out
  Page transitions: 300ms ease-in-out
  Loading spinner: 1s linear infinite
  Count-up numbers: 800ms cubic-bezier(0.33, 0.66, 0.66, 1)

=== MOBILE RESPONSIVENESS ===

- Cards: Full width with 16px padding

- Font sizes: Scale down by 10-15% on mobile

=== IMPLEMENTATION NOTES ===

1. Use CSS variables for colors (so dark/light mode works globally)
2. All transitions: Use cubic-bezier(0.4, 0, 0.2, 1) for smoothness
3. Test animations on lower-end devices (shouldn't be janky)
4. Use transform + opacity for animations (GPU accelerated)
5. Avoid box-shadow changes on hover (use transform: scale instead)
6. Loading states: Show skeleton for 500ms minimum (feels more real)
